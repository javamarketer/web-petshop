================================================================================
                          HOW TO GET STARTED
================================================================================

QUICKEST WAY (5 minutes):

1. Setup Database:
   mysql -u root -e "CREATE DATABASE petshop_jogja CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

2. Setup Backend:
   cd backend
   npm install
   npm run migrate
   npm run seed
   npm run dev

3. Setup Frontend (new terminal):
   cd frontend
   npm install
   npm run dev

4. Open http://localhost:3000 ✅

================================================================================
                         IMPORTANT REMINDERS
================================================================================

1. KEEP BOTH SERVERS RUNNING
   - Terminal 1: Backend on port 5000
   - Terminal 2: Frontend on port 3000
   - Keep both running while developing

2. CHECK ENVIRONMENT VARIABLES
   - backend/.env (database config)
   - frontend/.env.local (API URL)

3. DATABASE MUST BE RUNNING
   - MySQL service must be started
   - Database must be created before migrations

4. CLEAR CACHE IF NEEDED
   - Browser cache (Ctrl+Shift+Delete)
   - npm cache (npm cache clean --force)
   - Next.js cache (rm -r .next/)

5. CHECK BROWSER CONSOLE
   - F12 to open DevTools
   - Check Console tab for errors
   - Check Network tab for API calls

================================================================================
                         QUICK TROUBLESHOOTING
================================================================================

"Port 5000 already in use"
→ Change PORT in backend/.env or kill existing process

"Cannot connect to database"
→ Verify MySQL is running and credentials in .env are correct

"API calls failing"
→ Check backend is running, verify frontend .env.local has correct URL

"Module not found"
→ Run npm install in the affected directory

"Port 3000 already in use"
→ Change port in next.config.js or kill process on port 3000

For more troubleshooting, see SETUP_AND_TEST.md
